# Go语言main()与init()函数
## 一、概述
main() 和 init() 是Go语言预留的两个特殊函数，分别承担程序入口和包初始化的核心职责，均具备无参数、无返回值的特性，且无需显式调用。

## 二、main()函数
### 1. 核心定位
- 可执行程序的**唯一入口**，程序启动后自动触发调用，无需手动调用。
- 必须位于 `main` 包中（`package main`），这是可执行程序的强制要求。

### 2. 关键特性
- 无参数、无返回值，函数签名固定为 `func main()`。
- 每个可执行程序**必须包含且仅能包含一个** `main` 包和 `main()` 函数，否则无法编译执行。
- 作用：承载程序的核心业务逻辑，可调用其他包的函数、处理数据、实现功能等。

## 三、init()函数
### 1. 核心定位
- 包初始化的专用函数，用于完成包级别的初始化操作，在包被导入时自动执行。

### 2. 关键特性
- 无参数、无返回值，函数签名固定为 `func init()`，且**不能被程序显式引用或调用**。
- 支持在同一个包（或同一个程序）中定义**多个 init() 函数**。
- 执行顺序：多个 init() 函数按“定义顺序”执行；若分布在多个文件中，按文件名的字典序（字母顺序）执行。
- 执行时机：**始终在 main() 函数之前执行**，与 main() 函数无依赖关系。

### 3. 核心用途
- 初始化无法在全局上下文直接初始化的全局变量（如需要复杂逻辑计算、依赖外部资源的全局变量）。

## 四、核心区别与执行流程
| 特性                | main()函数                  | init()函数                  |
|---------------------|-----------------------------|-----------------------------|
| 所属包              | 必须在 main 包              | 可在任意包中                |
| 执行时机            | 程序启动后，所有init执行完毕 | 包导入时，main执行前        |
| 数量限制            | 每个可执行程序仅1个         | 同一包可多个                |
| 核心作用            | 程序入口，承载业务逻辑      | 包初始化，处理全局变量初始化|
